# 账号与权限预配置（上线前）

三级角色：**负责人（leader）** → **主管（supervisor）** → **专员（specialist，仅小程序）**；另有 **副负责人（delegated leader）**，由主负责人授权。

## 1. 主负责人（环境变量）

- 在部署环境（如 ECS `audit-admin.env`）中配置 `LEADER_ACCOUNTS`、`PRIMARY_LEADER_PHONE`（如有）等，与 `README.md` 一致。
- 主负责人使用 **手机号 + 密码** 登录 PC：`/reviews/login`。

## 2. 上线前建议操作（负责人登录 PC）

1. 登录后访问 **账号管理** `/users`。
2. **添加主管**：填写姓名、手机号；初始密码一般为手机号，首次登录后应要求修改强密码。
3. **添加副负责人**（仅主负责人可见）：与主管类似，权限与负责人接近，但「授权副负责人」仅主账号可用。
4. 用 **主管账号** 退出后重新登录，确认：
   - 可访问：复核池、问答日志、知识库；
   - 不可访问：账号管理（非负责人）。

## 3. 专员（小程序）

- 专员 **不分配 PC 密码**，通过 **微信登录** 自动注册为 `specialist`。
- 专员在 PC 无入口；仅可在小程序中提问与查看「我的复核」。

## 4. 备案期间

- 若公网域名无法访问，仅在本地或内网联调时，仍可用上述流程在 **本地后台** 预创建主管账号，备案通过后再让主管使用公网地址登录。

## 5. 安全提示

- 勿在仓库中提交真实 `.env`、密码或密钥。
- 生产环境务必设置强 `JWT_SECRET` 并定期轮换（需同步所有实例）。
