# 本地全链路测试清单（ICP 备案等待期）

不依赖域名备案，在本地即可完成「小程序 → 后台 → 复核 → 知识沉淀」验证。

## 环境准备

1. **后台**：仓库根目录执行 `npm install`，再 `npm run dev:admin`，浏览器打开 `http://127.0.0.1:3003`。
2. **环境变量**：复制 `apps/admin/.env.example` 为 `apps/admin/.env.local`，填入 `KV_REST_*`（或本地开发可用的 Redis）、`DASHSCOPE_API_KEY`、`JWT_SECRET`、`WX_APPID` / `WX_APP_SECRET`（测微信登录时需要）等。
3. **小程序**：微信开发者工具打开 `apps/miniprogram`，**详情 → 本地设置** 勾选 **不校验合法域名**；在设置页将 API 指向 `http://127.0.0.1:3003`（或本机局域网 IP + 端口）。

## 测试步骤（勾选即完成）

### A. PC 负责人 / 主管

- [ ] 访问 `/reviews/login`，使用负责人或主管账号登录成功。
- [ ] 首页 `/` 显示复核统计与快捷入口。
- [ ] **知识库** `/knowledge`：切换四个 Tab，能加载数据；**下载模板**、**Excel 导入**（可用 `npm run validate:knowledge-excel` 先校验表头）。
- [ ] **问答日志** `/conversations`：能看到记录列表；筛选标签正常。
- [ ] **复核池** `/reviews`：列表与进入详情 `/reviews/[id]` 正常。
- [ ] **负责人**：`/users` 创建主管、副负责人（若主账号）、停用/启用、改密码流程可用。

### B. 小程序专员

- [ ] 启动后自动或手动完成登录，拿到 token（控制台无持续报错）。
- [ ] **常规问题**：提交一条问题，能返回结果或进入「待复核」类状态。
- [ ] **我的复核**：能看到与 PC 侧一致的复核任务（按 requester 过滤）。

### C. 闭环

- [ ] PC **问答日志** 中出现小程序刚产生的记录。
- [ ] 主管将「AI 已自动回答」标为答错（若适用）后，任务进入待复核。
- [ ] 在 **复核详情** 填写人工回复并保存；必要时 **保存并加入知识库**。
- [ ] 再次在知识库中确认对应条目或规则已更新（视业务逻辑而定）。

### D. 回归

- [ ] 执行 `npm run verify:semantic-cases`（若已配置向量与用例）。
- [ ] 执行 `npm run test`（自动化测试，见根目录 `package.json`）。

## 常见问题

- **本机小程序连不上后台**：检查防火墙、是否同一网段、是否使用 `127.0.0.1`（真机需改电脑局域网 IP）。
- **401 / 未登录**：PC 页先登录；API 带 Basic 或 Cookie 与中间件一致。
- **表头导入失败**：使用 `npm run validate:knowledge-excel` 对照模板列名。
