# 数据目录

存放稽核规则的结构化数据模板与说明。线上环境默认使用 **Redis** 中的知识库；本地无 Redis 时可回退读取本目录 CSV。

## 模板文件（`data/templates/`）

| 文件                    | 对应后台表名         | 说明         |
| ----------------------- | -------------------- | ------------ |
| `02_共识解释表.csv`     | `consensus`          | 共识解释     |
| `03_常规问题规则表.csv` | `rules`              | 常规问题规则 |
| `04_旧品清单表.csv`     | `old-items`          | 旧品清单     |
| `05_外购清单表.csv`     | `external-purchases` | 外购清单     |

建议整理顺序与业务依赖一致：先共识与规则，再清单类。

## Excel 导入（推荐）

PC 端 **知识库** 页面支持 `.xlsx` 批量导入。导入前请从后台点击 **下载模板**，按模板列名填写，避免列名不一致导致写入失败。

### 本地校验表头（不启动服务）

在仓库根目录执行（需已 `npm install`）：

```bash
npm run validate:knowledge-excel -- path/to/你的文件.xlsx [表名]
```

表名可选：`rules` | `consensus` | `external-purchases` | `old-items`。若省略，会尝试根据文件名关键字推断。

若校验失败，终端会列出 **缺少的列名**，请与模板对齐。

## 语义 / 向量

规则更新并上线后，若启用 Qdrant，可执行根目录脚本同步向量（见根目录 `README.md`）：

```bash
npm run sync:rule-embeddings
```
